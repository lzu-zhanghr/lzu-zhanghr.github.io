markdown规范

Markdown 编写规范

采用 GFM, 以及百度 fex-team 团队规范格式编写所有 markdown 文件。

文件规范

  • 文件后缀必须使用 .md
  • 文件名必须使用小写
  • 文件编码必须使用 UTF-8
  • 文件标题书写采用如下规范, 使用 26个 = 号隔开
Markdown 规范
==========================

章节规范

  • 章节标题必须以 ## 开始
  • 章节标题与内容之间必须包含一个空行

表格规范

  • 使用 - 和 | 符号创建表格, 表格与内容之间插入空行
第一行 第二行
内容1 内容2
内容3 内容4 
第一行 | 第二行 
----- | ----- 
内容1 | 内容2 
内容3 | 内容4

代码规范

  • 代码块插入使用连续两个 ```
  • 代码块中插入 ``` 代码时使用 ````代替
  • 代码块与文本之间插入空行
``` markdown [文本块](超链接)
## 演示使用
```

正文规范

  • 英文和数字使用半角字符
  • 中文文字之间不使用空格
  • 中文文字与英文、阿拉伯数字及 @ $ % ^ & * . () 之间使用空格
  • 中文标点之间不加空格
  • 中文标点与前后字符之间均不加空格
  • 若括号内有中文，使用中文括号
  • 若括号内全部为英文，则使用半角英文符号
  • 使用直角引号 (「」) 代替双引号
