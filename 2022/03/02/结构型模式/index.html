结构型模式

结构型模式

适配器模式

  • 内容：将一个类的接口转换成客户希望的另一个接口，使由于接口不兼容而无法共同工作的类可以协作
  • 实现方式：
    • 类适配器：多继承
    • 对象适配器：组合
from abc import ABCMeta, abstractmethod

class Payment(metaclass=ABCMeta):
    @abstractmethod
    def pay(self, money):
        pass

class Alipay(Payment):
    def pay(self, money):
        print(f"支付宝支付{money: d}元")

class WechatPay(Payment):
    def pay(self, money):
        print(f"微信支付{money: d}元")

class BankPay:
    def cost(self, money):
        print(f"银联支付{money: d}元")

# 使用多继承实现适配器，使接口保持了一致

class InheritBankPay(Payment, BankPay):
    def pay(self, money):
        self.cost(money)

p1 = InheritBankPay()
p1.pay(200)

# 使用组合类实现适配器，使接口保持了一致

class CompositeBankPay(Payment):
    def __init__(self, payment):
        self.payment = payment

    def pay(self, money):
        self.payment.cost(money)

p2 = CompositeBankPay(BankPay())
p2.pay(100)

桥模式

  • 内容：将一个事物的两个维度分离，使其可以独立的变化, 可以很便利的在不同维度添加对象
  • 优点：
    • 抽象和实现相互分离
    • 优秀的扩展能力

组合模式

  • 内容：将对象组合成树形结构以表示“部分-整体”的层次结构，使得单个对象和组合对象的使用具有一致性。
  • 优点：
    • 定义了包含基本对象和组合对象的层次结构
    • 简化了客户端代码，使客户端可以一致使用组合对象和单个对象
    • 更容易增加新类型的组件
from abc import ABCMeta, abstractmethod

class Graphic(meataclass=ABCMeta):

外观模式

  • 内容：为子系统中的一组接口提供一个一致的界面，外观模式定义了一个高层接口，使得这一子系统更加容易使用。
    class CPU:
    def run(self):
        print("cpu is running")

    def stop(self):
        print("cpu stop run")

class Disk:
    def run(self):
        print("disk is running")

    def stop(self):
        print("disk stop run")

class Memory:
    def run(self):
        print("memory is running")

    def stop(self):
        print("memory stop run")

class Computer:
    def __init__(self):
        self.cpu =  CPU()
        self.disk = Disk
        self.memory = Memory
    
    def run(self):
        self.cpu.run()
        self.disk.run()
        self.memory.run()

    def stop(self):
        self.cpu.stop()
        self.disk.stop()
        self.memory.stop()

computer = Computer()
computer.run()
computer.stop()

代理模式

  • 内容：为其他对象提供一种代理以控制对这个对象的访问
  • 优点：
    • 实代理:可以隐藏对象位于远程地址空间的事实
    • 虚代理:可以进行优化，节省不必要的系统开销
    • 保护代理：允许访问对象时进行附加处理
from abc import ABCMeta, abstractmethod

class Subject(metaclass=ABCMeta):
    @abstractmethod
    def get_content(self):
        pass
    
    @abstractmethod
    def set_content(self, content):
        pass

class RealProxy(Subject):
    def __init__(self, filename):
        self.filename = filename
        with open(filename, "r") as f:
            self.content = f.read()
            
    def get_content(self):
        return self.content
    
    def set_content(self, content):
        with open(self.filename, 'w') as f:
            f.write(content)

# 虚代理实现内存节省

class VirtualProxy(Subject):
    def __init__(self, filename):
        self.filename = filename
        self.content = None

    def get_content(self):
        if not self.content:
            with open(filename, "r") as f:
                self.content = f.read()
        return self.content 

    def set_content(self, content):
        with open(self.filename, 'w') as f:
            f.write(content)
